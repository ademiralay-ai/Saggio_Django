1) Sistem Mimarisi

Saggio RPA merkezi bir orkestrasyon yapisidir. Django uygulamasi is dagitim merkezi gibi davranir; robot ajanlari kendi makinelerinde calisir ve merkezden is cekerek gorevleri tamamlar.

  • Django Merkez: Surec tanimlari, adimlar, kuyruk, scheduler, release ve log kayitlari.
  • Robot Ajan: Heartbeat atar, bekleyen isi ceker, calistirir, sonuc raporlar.
  • Flow Builder: SAP ve kontrol adimlarini gorsel olarak olusturur.
  • Notification Katmani: Telegram, Mail ve FTP ile entegrasyon kurar.
  • Operational UI: Robot Operasyon ekranindan canli is emri ve release dagitimi yapilir.

5) Sifirdan Surec Olusturma Rehberi

  1. Surec kaydi ac: SAP Surecleri ekraninda yeni surec ekleyin, isim ve temel runtime parametrelerini girin.
  2. Builder ekranini ac: Surece girip gorsel editoru acin; Start, SAP adimlari, IF/ELSE, LOOP ve End node yerlesimi yapin.
  3. Node konfiglerini doldur: Her adimda zorunlu alanlari doldurun (template, komut, bekleme kriteri, degisken vb.).
  4. Baglantilari kur: Kontrol akisinda true/false ve continue/done cikislarini dogru node'lara baglayin.
  5. Kaydet ve test et: Once adim modu (step mode), sonra otomatik calisma ile dry run yapin.
  6. Run preview ve runtime ayari: Sureci ilgili robotta calisacak sekilde timeout/retry ayarlariyla yayinlayin.
Kritik tasarim kurallari
  • Her surecte tek bir Start ve en az bir End olmasi tavsiye edilir.
  • Kosul adimlarinda true/false kollarinin ikisi de bos kalmamali.
  • Dongu adimlarinda done kolu cikis yoluna baglanmali; aksi halde surec kilitlenebilir.
  • Uretime cikmadan once ayni data ile en az 3 tekrarli test yapilmalidir.

6) Robot Ekleme ve Ajan Kurulumu

Adım 1 — Sunucu tarafı: yeni ajan kaydı

Merkez sunucuda yeni robot ajanını oluşturun. Bu komut bir token üretir; sonraki adımlarda gerekecek.

c:\Util\SaggioDjango\.venv\Scripts\python.exe `
    c:\Util\SaggioDjango\manage.py create_robot_agent `
    --code robot-06 --name "Robot 06"
c:\Util\SaggioDjango\.venv\Scripts\python.exe ^
    c:\Util\SaggioDjango\manage.py create_robot_agent ^
    --code robot-06 --name "Robot 06"

Önemli: Komutun çıktısındaki token değerini saklayın — bir daha gösterilmez.

Adım 2 — Kurulum paketini hazırla

Sunucu üzerinde aşağıdaki script paketleyiciyi çalıştırın; dist\SaggioRobotAgentPackage_X.Y.Z.zip üretilir.

cd c:\Util\SaggioDjango
.\build_robot_agent_package.ps1                    # internet varsa NSSM kurulumda inilir
.\build_robot_agent_package.ps1 -IncludeNssm       # offline kurulum icin NSSM'i pakete goml

Adım 3 — Robot makinesine paketi gönder ve aç

  1. ZIP'i robot makinesine kopyalayın (USB / paylaşılan klasör / RDP).
  2. Örnek olarak C:\Users\admin\Desktop\ altına çıkartın.
  3. Klasörde PowerShell'i Administrator olarak açın.

Adım 4 — Tek komutla kurulum

Aşağıdaki komutu kendi ServerUrl, AgentCode ve Token değerlerinizle çalıştırın. Script venv kurulumu, bağımlılıklar, config.json üretimi, NSSM ile servis kaydı dahil her şeyi otomatik halleder.

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force

.\install.ps1 -ServerUrl "http://10.16.88.18:8000" `
              -AgentCode "robot-06" `
              -Token     "SUNUCUDA_OLUSAN_TOKEN" `
              -Name      "Robot 06"
powershell -ExecutionPolicy Bypass -File install.ps1 ^
    -ServerUrl "http://10.16.88.18:8000" ^
    -AgentCode "robot-06" ^
    -Token     "SUNUCUDA_OLUSAN_TOKEN" ^
    -Name      "Robot 06"

Kurulum bittiğinde SaggioRobotAgent Windows servisi otomatik başlar ve auto-start olarak kayıtlıdır (yeniden başlatmada da çalışır).

Adım 5 — Doğrulama

Get-Service SaggioRobotAgent
Get-Content C:\SaggioRobotAgent\logs\launcher.log -Tail 30
Get-Content C:\SaggioRobotAgent\logs\stdout.log   -Tail 30
Test-NetConnection 10.16.88.18 -Port 8000

Sunucudaki Robot Operasyon ekranında robotun durumu Online olmalı.

Güncelleme

Yeni paketi açıp aynı install.ps1 komutunu tekrar çalıştırın; eski servis temizlenip yenisi kurulur. Mevcut config.json otomatik config.json.bak olarak yedeklenir.

Kaldırma

.\uninstall.ps1                    # sadece servisi kaldirir, veri kalir
.\uninstall.ps1 -PurgeData         # her seyi siler (config + log dahil)

7) Operasyon Akisi: Is Dagitimi, Scheduler, Release

Canli is dagitimi (dispatch)

  • Robot Operasyon ekranindan hedef robotu secin.
  • command_type ve payload tanimini girin.
  • Oncelik (priority) ile kuyruktaki siralamayi belirleyin.

Scheduler ile otomasyon

  • Surec + robot + zamanlama tanimlanir.
  • Bakim penceresi ve cakisma (overlap) kurallari belirlenir.
  • Run-now ile manuel tetikleme yapilarak plan test edilir.

Release ve versiyon dagitimi

  • Yeni agent release kaydi acilir (version, url, checksum, install command).
  • Desired version tek ajan veya tum ajanlara atanir.
  • Ajanlar heartbeat/check-update dongusunda eskiyse outdated olarak gorunur.
Onemli: Is guvenligi

Uretim robotunda ayni anda birden cok agir surec tetiklemek kaynak darbogazi yapabilir.

Onemli: Versiyonlama

Zorunlu (mandatory) release tanimlarken rollback paketi hazir olmadan yayina cikmayin.

Onemli: Gozlemlenebilirlik

Her kritik surecin basari ve hata bildirimini Telegram/Mail ile aktif tutun.

8) API ve Teknik Calisma Mantigi

Robot ajan-merkez iletisiminde temel akisin ozeti:

  1. Register: Robot ilk acilista kimlik dogrulama ile kaydolur.
  2. Heartbeat: Belirli araliklarla canli oldugunu bildirir.
  3. Pull job: Kuyruktan kendisi icin bekleyen isi ceker.
  4. Run process: Isi yerelde calistirir, ara log/event yollar.
  5. Job update: Basari, hata, iptal, sure gibi sonuc metriklerini merkeze yazar.
Temel endpoint aileleri
  • /api/robot-agent/register, /heartbeat, /pull-job, /job-update
  • /api/robot-agent/dispatch-job, /jobs, /events, /status
  • /api/scheduler/list, /save, /toggle, /run-now, /dispatch-due
  • /sap-process/.../flow/* ve scan endpointleri

9) Sadece Admin: Canli Operasyon Proseduru

Bu bolum yalnizca admin kullanicilara gosterilir. Uretim aninda canli operasyonu durdurmadan risk yonetimi yapmaniz icin adim adim prosedur sunar.

  1. Durum anlik tespiti: Dashboard + Robot Operasyon ekranindan online ajan, job backlog ve hata artis hizini kontrol et.
  2. Etkilenen kapsam: Hangi process_id, hangi agent_code ve hangi zaman araliginda bozulma oldugunu ayikla.
  3. Hasar sinirlama: Sorunlu sureci scheduler'da pasife al; kritik olmayan dispatch'leri gecici olarak durdur.
  4. Yuk dengeleme: Uygunsa hedef ajanlari degistirip islemleri saglikli robotlara dagit.
  5. Iletisim: Is birimine etki suresi, beklenen toparlanma zamani ve gecici workaround bilgisini gec.
  6. Kalici duzeltme: Hata kok nedenine gore config/runtime/release duzeltmesi yapip kademeli geri acilis uygula.
  7. Kapanis raporu: Olay kaydi ac, once/sonra metriklerini ekle, tekrar onleme aksiyonlarini planla.
Admin acil durum checklist
  • Robot servisleri calisiyor mu?
  • Token veya desired version uyusmazligi var mi?
  • Ayni surec birden fazla scheduler kaydiyla cakisiyor mu?
  • Dis servis (SAP/FTP/Telegram/Mail) gecikmesi var mi?

10) Sadece Developer: Endpoint + Veri Modeli Referansi

Bu bolum yalnizca superuser (developer) kullanicilara gosterilir. Hizli gelistirme, debug ve entegrasyon icin cekirdek endpoint ve model iliskilerini ozetler.

Endpoint gruplari

  • Agent Runtime: /api/robot-agent/register, /heartbeat, /pull-job, /job-update, /check-update, /log-event
  • Robot Admin: /api/robot-agent/status, /jobs, /events, /dispatch-job, /cancel-job, /upsert
  • Release: /api/robot-agent/releases/*, /set-desired, /deploy, /build-setup, /build-package
  • Scheduler: /api/scheduler/list, /save, /toggle, /delete, /run-now, /dispatch-due
  • SAP Process: /sap-process/<id>/... scan/runtime/flow endpointleri

Veri modeli cekirdegi

  • RobotAgent: Robot kimligi, token, istek surumu, aktiflik ve heartbeat durumu.
  • RobotJob: Kuyruklanan gorev, command_type, payload, durum ve sonuc metrikleri.
  • RobotAgentRelease: Agent surumu, dagitim URL'i, setup/install metadata.
  • SapProcess: Is sureci ana tanimi, akis baglantilari ve runtime parametreleri.
  • PeriodicProcessSchedule: Planli tetikleme kurallari, overlap ve bakim penceresi.
Developer debug akisi
  1. Hatali istekte endpoint request/response ciftini izole et.
  2. Ilgili model kaydini id/time bazli eslestir.
  3. Agent event stream ile server-side job state gecislerini karsilastir.
  4. Stale runtime durumunda sureci stop edip temiz heartbeat dongusuyle yeniden baslat.

11) Sorun Giderme Rehberi

Hizli kontrol tablosu

  • Diger PC acamiyor: runserver 0.0.0.0:8000 ve firewall inbound kurali kontrol edin.
  • Robot offline: config token/agent_code/server_url uyumu ve servis durumu kontrol edin.
  • Job pending kaliyor: Hedef robot enabled mi, pull-job donuyor mu kontrol edin.
  • PS1 calismiyor: ExecutionPolicy ve Unblock-File uygulayin.
  • SAP adimi sapti: Tarama verisini guncelleyip step-mode ile tekrar test edin.
Operatorlar icin 5 dakikalik acil durum akisi
  1. Dashboard ve Robot Operasyon ekranlarinda online/queue durumunu kontrol et.
  2. Hata veren isi job listesinden ac ve hata mesajini not al.
  3. Ayni isi dusuk oncelik yerine normal oncelikte yeniden dene.
  4. Tekrarlayan hata varsa ilgili sureci adim modu ile test ortaminda calistir.
  5. Cozulmezse admin/developer ekibine process_id, agent_code, timestamp ile aktar.
